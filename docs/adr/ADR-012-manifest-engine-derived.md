# ADR-012: MANIFEST.md is an engine-derived view, regenerated on every `state save`

**Status:** Accepted
**Date:** 2026-06-08

## Context

`.sovereign/MANIFEST.md` is loaded FIRST by every skill, every session, as the
agent's orientation document, and must stay under ~500 tokens (SOVEREIGN ADR-006).
A hand-maintained MANIFEST inevitably goes stale: contributors update STATE.md,
decisions, or blockers and forget to mirror them into MANIFEST, so the
first-loaded orientation document silently lies. Research flagged this as a concrete
pitfall (stale MANIFEST). The fix is to make MANIFEST a *derived* artifact, never a
hand-edited one.

## Decision

`MANIFEST.md` is an **engine-derived view**:

- It is **regenerated by `sovereign-tools` on every `state save`** from the
  authoritative sources (STATE.md fields, decisions, stack, blockers,
  next-recommended-action).
- It is **never hand-edited.** The template carries a header comment saying so, and
  the engine owns its contents.
- The **token budget (~500 tokens) is enforced in code** on save — the engine counts
  tokens (a chars/4 heuristic is acceptable) and truncates the quick-reference tables
  to top-N (or warns) so the budget cannot be exceeded.

This is the one place SOVEREIGN deliberately extends GSD's state mechanics: GSD's
`state save` patches fields; SOVEREIGN's also re-derives MANIFEST.

## Consequences

- **Positive:** The first-loaded orientation document is always accurate and always
  within budget — no stale-MANIFEST drift, no token blowups on session start.
- **Positive:** Contributors have one less file to maintain by hand; correctness is
  guaranteed by code, not discipline.
- **Negative / accepted cost:** Manual edits to MANIFEST.md are overwritten on the
  next `state save` — by design. Anything that must persist belongs in an
  authoritative source (STATE.md, config.json, ADRs), not in MANIFEST.
- **Implementation note:** The exact token-counting heuristic and truncate-vs-warn
  strategy are left to implementation discretion (chars/4 approximation acceptable).
