<objective>
Hand-author `engine/skills/stack-select/SKILL.md` — the guided, recommendation-first technology-stack-selection skill. It gathers the inputs that actually determine the choice, one at a time, recommending as it goes — project type/track (web/mobile/backend/data/etc.), scale target (launch + horizon), budget (asked explicitly: infra + team), team/skill context, and hard constraints (regulatory, region, existing systems, agent-friendliness). For each layer (language/runtime, framework, datastore, infra primitives) it recommends the BEST tool for THESE constraints with reasoning and what it is NOT picking and why — explicitly NOT trend-following. It records the chosen stack + rationale to `.sovereign/docs/STACK.md` (updating in place on re-run), OFFERS `/adr-log` for lock-in choices (datastore/framework/cloud) that pass the three-condition gate rather than writing ADRs itself, and — when a recommendation depends on fast-moving facts (current versions, pricing, new managed options) — SAYS SO and flags that the user may want to verify against current docs (the future `anchor-docs`, M3) instead of asserting stale specifics with false confidence.

Purpose: Establish a recorded, constraint-driven stack choice (not vibes, not trend-following) that downstream construction implements against, and prove the adr-log composition once more. Closes ARCH-04 and (for this skill) ARCH-08.
Output: One shipped skill (`engine/skills/stack-select/SKILL.md`; `--full` install only, NOT in FAST_LANE).

  itself; adr-log is the single owner of ADR recording.

  discretion — fine to read by path, degrade gracefully if absent.
