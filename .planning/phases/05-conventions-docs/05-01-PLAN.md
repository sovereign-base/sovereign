<!-- Verified against the codebase — author against these, do not re-derive. -->

REAL Agent Skills frontmatter fields actually used by the 6 skills + advisor agent:
- `name` (every skill) — ≤64 chars, lowercase-hyphen, no "claude"/"anthropic" (enforced by `sovereign-tools validate skills`).
- `description` (every skill) — leads with the trigger use-case; within the 1024-char API cap / 1536 listing cap.
- `disable-model-invocation: true` (council only) — for user-only / side-effecting / orchestrator skills; removes the skill from the listing budget.
- `argument-hint` (council) — e.g. "[--standard] \"<decision>\"".
- Agent-only fields (sovereign-advisor.md): `tools`, `model: inherit` (engine resolves via model-profiles), `color`.

v1 NON-STANDARD fields to DROP (seen in archive/v1/skills/fast-lane/grill-with-docs/SKILL.md), each with WHY:
- `triggers` — not in the Agent Skills spec; triggering is driven by `description` + `when_to_use`.
- `works-best-with` / `compatible-with` — non-standard; portability hints belong in namespaced `metadata:` if anywhere.
- `min-model` — non-standard; model selection lives in the engine (model-profiles), not frontmatter.
- `tokens` — non-standard; budget is engine-enforced via `doctor`, not declared.
- bare `phase:` — non-standard; phase logic lives in the engine, not frontmatter.
(These five are exactly the fields ADR-011 "drop v1 frontmatter" locks out.)

The thin-body / single-init-load rule (every Phase 3-4 skill obeys it):
- A skill orients with ONE `sovereign-tools init <skill>` call and the `@file:` spill guard:
  `INIT=$(node "$ENGINE/bin/sovereign-tools.cjs" init <skill>); if [[ "$INIT" == @file:* ]]; then INIT=$(cat "${INIT#@file:}"); fi`
- The skill performs ZERO other orientation reads — `init` already resolved models, config, phase, and paths. Heavy detail lives in referenced files (progressive disclosure). "Thinness IS delegation to init."
- All state/gate/commit/model side-effects are delegated to `sovereign-tools` (state save, gate pass, commit) — the SKILL.md holds flow + content only.

Required body sections (present across all 6 skills):
- `## Why this matters` — plain-language (Profile-A), the cost-of-not-doing-it hook.
- `## When to use this` — including explicit anti-uses ("Don't use it for…").
- The flow — numbered steps, step 1 always the single `init` orient call, side-effects delegated to the engine.
- A navigation footer (`## Navigation`) — recommendation-first ("▶ NEXT" with the recommended action first, alternatives after).

Listing-budget rule (already documented at engine/references/listing-budget.md):
- Auto-triggerable skills count against ~1% (~2000 token) budget; orchestrator-only/side-effecting skills set `disable-model-invocation: true` to stay OFF the budget. Enforced by `sovereign-tools validate skills` (frontmatter lint) + `sovereign-tools doctor` (budget count, AUTO_MAX≈7).
