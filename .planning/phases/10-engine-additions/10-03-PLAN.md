  <files>engine/bin/lib/adopt.cjs</files>
  <behavior>
    - scanProject(cwd) on a tmp project with package.json + a Dockerfile + a src/ tree → manifests includes {path:'package.json', kind:'npm', present:true}; detected.has_dockerfile true; detected.languages includes a value inferred from manifests/extensions; structure.tree lists filenames only (no contents) and structure.file_count > 0.
    - Gitignore-aware: when cwd is a git repo, tree comes from `git ls-files` (so .gitignore'd / node_modules entries are absent); when NOT a git repo, a bounded manual walk skipping node_modules/.git/dist produces the tree.
    - Cap: when file count exceeds the cap (e.g. 2000), structure.tree is truncated to the cap and structure.truncated === true; otherwise truncated === false.
    - deep_read_candidates contains heuristic {path, reason} entries when matching files exist (entrypoint: src/index.*/main.*/app.*; auth: middleware/auth*; base-model: models/ or schema.*; config: the primary manifest/config), else [].
    - Greenfield-safe: scanProject on an empty non-git tmp dir → all arrays empty, all flags false, file_count 0, truncated false, NEVER throws. Never writes to cwd.
  </behavior>
  <action>
    Create `engine/bin/lib/adopt.cjs` (header + JSDoc describing the ADOPT Layers-1+2 substrate + the read-only scope guard). Require `node:fs`, `node:path`, and `{ output, execGit, safeReadFile }` from `./core.cjs`.

    Layer 1 — manifest detection: define a table mapping filename → kind (package.json→npm, pyproject.toml/requirements.txt→pip, go.mod→go, Cargo.toml→cargo, pom.xml/build.gradle→maven-gradle, Gemfile→bundler, composer.json→composer, Dockerfile→docker, docker-compose.yml→compose, .env.example→env, tsconfig.json→typescript). For each, `present: fs.existsSync`. Build `manifests` (present ones). Infer `detected.languages` (typescript if tsconfig/.ts present, javascript if package.json, python if pyproject/requirements, go/rust/ruby/php/java from their manifests) and `package_managers` from the same. Flags: `has_dockerfile` (Dockerfile present), `has_ci` (`.github/workflows/` exists), `has_tests` (a `test`/`tests`/`__tests__` dir or `*.test.*`/`*.spec.*` present), `monorepo` (workspaces in package.json OR a `packages/`/`apps/` dir).

    Layer 2 — structure: get the file list via `execGit(cwd, ['ls-files'])` when `execGit(cwd, ['rev-parse', '--is-inside-work-tree']).exitCode === 0` (split stdout on newlines); ELSE a bounded recursive `fs.readdirSync` walk skipping `node_modules`, `.git`, `dist`, `build`, `.next`, vendored dirs. Cap the tree at a constant `MAX_TREE` (e.g. 2000); set `structure.truncated = fullList.length > MAX_TREE`; `structure.file_count = fullList.length`; `structure.tree = fullList.slice(0, MAX_TREE)`; `structure.top_level_dirs` = unique first path segments that are dirs.

    deep_read_candidates: heuristic scan of the tree for entrypoint/auth/base-model/config matches → `[{path, reason}]` (cap ~10).

    `scanProject(cwd)` returns the full contract object (pure read; never writes). `cmdAdoptScan(cwd, raw)` → `output(scanProject(cwd), raw)` (relies on @file: spill for large trees). Greenfield-safe everywhere. `module.exports = { cmdAdoptScan, scanProject }`. Zero new deps.
  </action>
